Through the DACA program I'm building real analytics skills from the ground up: SQL, Python, data cleaning, and visualization — and turning each week's work into a project worth showing.

---

## 🏢 The case: UrbanStyle.ltd

Every project here works with **UrbanStyle.ltd**, a (fictional) Estonian
sustainable-fashion company that grew 150% in two years — and now drowns in
messy data across three disconnected systems. My job across 11 weeks: clean it,
analyze it, and produce the insights the founder needs for her investor pitch.

The data lives in a PostgreSQL (Supabase) database with five core tables:
`sales` (~15k rows, with duplicates), `customers`, `products`, `inventory`,
and `web_logs`.

## 📂 How to read this portfolio

Each `week-N/` folder is one milestone with its own `README.md` that answers
three questions: **why** the project exists (the UrbanStyle business need),
**what** I did (data, approach, tool), and **what it shows** (the result in
business terms). Artifacts — SQL scripts, notebooks, dashboards, screenshots —
live alongside that README. The portfolio grows one folder per week.
